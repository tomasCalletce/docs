## Resumen

**VTEX Headless** es una arquitectura donde el frontend es una aplicación separada que se comunica con VTEX a través de APIs. Como tienes control total del código, la integración con Proof se hace capturando el Transaction ID en el frontend y reportando conversiones vía postback S2S.

<Info>
  Tu technical account manager te guía en todo el proceso de integración. Los ejemplos de código en esta página son referencia para tu equipo técnico.
</Info>

## Prerrequisitos

- Acceso al código fuente de tu frontend
- Acceso al **admin de VTEX** (para configurar notificaciones de órdenes)
- Una cuenta de marca activa en Proof
- Al menos una oferta creada en Proof

---

## Paso 1: Capturar el Transaction ID

Cuando un usuario llega desde un tracking link, la URL contiene un parámetro `tid` que identifica al creador. Necesitas capturarlo y guardarlo en una cookie.

En tu layout principal (el componente que carga en todas las páginas), agrega la lógica para capturar el parámetro:

```javascript
// Capturar el Transaction ID del tracking link y guardarlo en una cookie
var params = new URLSearchParams(window.location.search);
var efTid = params.get('tid');
if (efTid) {
  var expires = new Date(Date.now() + 30 * 24 * 60 * 60 * 1000).toUTCString();
  document.cookie = 'proof_tid=' + efTid + '; path=/; expires=' + expires;
}
```

Este código se ejecuta una vez al cargar la primera página. Si el usuario llegó desde un tracking link, captura el Transaction ID y lo guarda en una cookie con expiración de 30 días.

---

## Paso 2: Guardar el Transaction ID en la orden

Para que el postback funcione, el Transaction ID debe quedar guardado en los datos de la orden de VTEX. Esto se hace actualizando el campo `marketingData` del `orderForm` durante el checkout.

```javascript
// Durante el checkout, antes de completar la orden
var match = document.cookie.match(/(?:^|; )proof_tid=([^;]+)/);
var efTid = match ? match[1] : null;

if (efTid) {
  fetch(
    '/api/checkout/pub/orderForm/' + orderFormId + '/attachments/marketingData',
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        utmSource: 'proof',
        utmCampaign: efTid
      })
    }
  );
}
```

<Warning>
  Reemplaza `orderFormId` con el ID del orderForm actual. Tu technical account manager te ayuda a identificar el mejor momento para ejecutar esto en el flujo de tu checkout.
</Warning>

---

## Paso 3: Reportar la conversión (Postback S2S)

Configura tu servidor para recibir notificaciones de VTEX cuando se completa una orden. Cuando llega una notificación, tu servidor extrae el Transaction ID y envía la conversión a Proof.

### Configurar el webhook de VTEX

1. Ve a **VTEX Admin** > **Orders Management** > **Settings** > **Notification**.
2. Configura un endpoint en tu servidor que reciba la notificación de órdenes nuevas.

### Disparar el postback desde tu servidor

```javascript
// Disparar postback a Proof con los datos de la orden
const params = new URLSearchParams({
  nid: '3935',
  transaction_id: efTid,        // El Transaction ID guardado en marketingData.utmCampaign
  amount: String(orderTotal),    // Monto de la venta
  adv_event_id: orderId          // Número de orden
});

await fetch(`https://www.track.proof.nilho.com/?${params}`);
```

<Info>
  Recuerda que VTEX almacena los montos en **centavos**. Divide por 100 antes de enviar el `amount` a Proof.
</Info>

## Probar la integración

<Steps>
  <Step title="Genera un tracking link de prueba en Proof">
    Crea un tracking link con un creador de prueba para tu oferta.
  </Step>
  <Step title="Haz clic en el tracking link">
    Abre el link en un navegador en modo incógnito. Verifica que llegas a tu sitio.
  </Step>
  <Step title="Completa una compra de prueba">
    Completa el checkout con un producto de prueba.
  </Step>
  <Step title="Verifica en Proof">
    En **Reports** > **Conversions**, confirma que la conversión aparece con el monto y número de orden correctos.
  </Step>
</Steps>

## Solución de problemas

<Accordion title="El Transaction ID se pierde durante la navegación">
  Asegúrate de que la cookie `proof_tid` se está creando correctamente. Si tu frontend usa Server Side Rendering, la cookie podría no estar disponible en el primer render del servidor. El script de captura debe ejecutarse del lado del cliente.
</Accordion>

<Accordion title="Las conversiones no aparecen en Proof">
  1. Verifica que el webhook de VTEX está configurado y apuntando a tu servidor.
  2. Confirma que el Transaction ID se está guardando en `marketingData.utmCampaign` de la orden.
  3. Revisa los logs de tu servidor para confirmar que el postback se está enviando.
</Accordion>

<Accordion title="El monto no coincide">
  VTEX maneja montos en centavos. Si el valor en VTEX es `15000000`, divídelo por 100 para obtener `150000.00` antes de enviarlo a Proof.
</Accordion>
