## Resumen

**VTEX IO** (también llamado Store Framework o VTEX Cloud) es la arquitectura moderna de VTEX. Usa un sistema de **apps** y componentes. La integración con Proof se hace a través de una **Pixel App** para capturar los clics, y un postback S2S para reportar las conversiones.

<Info>
  Tu technical account manager te guía en todo el proceso de integración. Los ejemplos de código en esta página son referencia para tu equipo técnico.
</Info>

## Prerrequisitos

- Proyecto VTEX IO configurado con acceso al CLI (`vtex` CLI)
- Acceso al **admin de VTEX** (para configurar notificaciones de órdenes)
- Una cuenta de marca activa en Proof
- Al menos una oferta creada en Proof

---

## Paso 1: Capturar el clic con una Pixel App

Una Pixel App en VTEX IO es un componente que se ejecuta en todas las páginas. Cuando un usuario llega desde un tracking link, la Pixel App captura el **Transaction ID** y lo guarda en una cookie.

### Crear la Pixel App

```bash
vtex init
# Selecciona "pixel" como tipo de app
```

### Configurar el manifest

En `manifest.json`, configura los permisos para comunicarse con Proof:

```json
{
  "name": "proof-tracking",
  "vendor": "tuvendor",
  "version": "0.1.0",
  "builders": {
    "pixel": "0.x"
  },
  "policies": [
    {
      "name": "outbound-access",
      "attrs": {
        "host": "www.track.proof.nilho.com"
      }
    }
  ]
}
```

### Implementar la captura de clics

En `pixel/head.html`, agrega el SDK de Proof:

```html
<script src="https://www.track.proof.nilho.com/scripts/sdk/everflow.js"></script>
<script>
  EF.click({
    offer_id: EF.urlParameter('oid'),
    affiliate_id: EF.urlParameter('affid'),
    transaction_id: EF.urlParameter('_ef_transaction_id')
  });
</script>
```

### Instalar la Pixel App

```bash
# Desarrollo local
vtex link

# Publicar
vtex publish
vtex install tuvendor.proof-tracking@0.1.0
```

## Paso 2: Guardar el Transaction ID en la orden

Para que el postback funcione, el Transaction ID debe quedar guardado en los datos de la orden de VTEX. En VTEX IO, esto se hace actualizando el `marketingData` del `orderForm` durante el checkout.

En el pixel o en un componente del checkout, lee la cookie y pásala al `orderForm`:

```html
<script>
  var match = document.cookie.match(/(?:^|; )ef_tid=([^;]+)/);
  var efTid = match ? match[1] : null;

  if (efTid) {
    // Escuchar cuando el checkout está listo
    function handleMessages(event) {
      if (!event.data || !event.data.event) return;

      if (event.data.event.eventName === 'vtex:orderPlaced') {
        // El Transaction ID ya fue guardado en la cookie por el SDK
      }
    }
    window.addEventListener('message', handleMessages);
  }
</script>
```

<Warning>
  Tu technical account manager te ayuda a determinar el mejor punto para guardar el Transaction ID en el `orderForm` de VTEX IO, ya que esto varía según la configuración de tu tienda.
</Warning>

## Paso 3: Reportar la conversión (Postback S2S)

Configura un webhook en VTEX para que notifique a tu servidor cuando se completa una orden. Tu servidor lee la orden, extrae el Transaction ID y envía la conversión a Proof.

### Configurar el webhook

1. Ve a **VTEX Admin** > **Orders Management** > **Settings** > **Notification**.
2. Configura un endpoint en tu servidor que reciba la notificación de órdenes nuevas.

### Disparar el postback desde tu servidor

```javascript
// En tu servidor
const handleOrder = async (orderId) => {
  const order = await vtexApi.getOrder(orderId);
  const efTid = order.marketingData?.utmCampaign;

  if (efTid) {
    const params = new URLSearchParams({
      nid: '3935',
      transaction_id: efTid,
      amount: String(order.value / 100),
      adv_event_id: orderId
    });

    await fetch(`https://www.track.proof.nilho.com/?${params}`);
  }
};
```

<Info>
  VTEX almacena los montos en **centavos**. Divide por 100 antes de enviar a Proof. Ejemplo: `150000` centavos = `1500.00` en la moneda local.
</Info>

## Probar la integración

<Steps>
  <Step title="Verifica la Pixel App">
    Usa `vtex link` para probar localmente. Verifica que el SDK de Proof carga correctamente en todas las páginas.
  </Step>
  <Step title="Genera un tracking link de prueba en Proof">
    Crea un tracking link con un creador de prueba para tu oferta.
  </Step>
  <Step title="Haz clic en el tracking link">
    Abre el link en un navegador en modo incógnito. Verifica que llegas a tu tienda VTEX.
  </Step>
  <Step title="Completa una compra de prueba">
    Completa el checkout con un producto de prueba.
  </Step>
  <Step title="Verifica en Proof">
    En **Reports** > **Conversions**, confirma que la conversión aparece con el monto y número de orden correctos.
  </Step>
</Steps>

## Solución de problemas

<Accordion title="La Pixel App no se ejecuta">
  Verifica que la app está correctamente instalada con `vtex list`. Asegúrate de que el `manifest.json` tiene los builders y policies correctos.
</Accordion>

<Accordion title="Las conversiones no aparecen en Proof">
  1. Verifica que el webhook de VTEX está configurado y apuntando a tu servidor.
  2. Confirma que el Transaction ID se está guardando en `marketingData.utmCampaign` de la orden.
  3. Revisa los logs de tu servidor para confirmar que el postback se está enviando.
</Accordion>

<Accordion title="El monto no coincide">
  VTEX maneja montos en centavos. Si el valor en VTEX es `15000000`, divídelo por 100 para obtener `150000.00` antes de enviarlo a Proof.
</Accordion>
