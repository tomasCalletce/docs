## Resumen

**VTEX Legacy** (Portal/CMS) es la plataforma original de VTEX. Usa templates HTML y el checkout clásico. La integración con Proof se hace en dos partes: capturar el clic del creador en el frontend, y reportar la conversión desde tu servidor vía postback S2S.

<Info>
  Tu technical account manager te guía en todo el proceso de integración. Los ejemplos de código en esta página son referencia para tu equipo técnico.
</Info>

## Prerrequisitos

- Acceso al **CMS de VTEX** (para editar templates HTML)
- Acceso al **admin de VTEX** (para configurar notificaciones de órdenes)
- Una cuenta de marca activa en Proof
- Al menos una oferta creada en Proof

---

## Paso 1: Capturar el Transaction ID

Cuando un usuario llega desde un tracking link, la URL contiene un parámetro `tid` que identifica al creador. Necesitas capturarlo y guardarlo en una cookie para no perderlo cuando el usuario navega tu tienda.

En el CMS de VTEX, edita el **template del header** que se incluye en todas las páginas. Agrega antes del cierre de `</head>`:

```html
<script>
  // Capturar el Transaction ID del tracking link y guardarlo en una cookie
  var params = new URLSearchParams(window.location.search);
  var efTid = params.get('tid');
  if (efTid) {
    document.cookie = 'proof_tid=' + efTid + '; path=/; max-age=' + (30 * 24 * 60 * 60);
  }
</script>
```

Este script se ejecuta en cada página. Si el usuario llegó desde un tracking link, captura el Transaction ID y lo guarda en una cookie con expiración de 30 días.

## Paso 2: Guardar el Transaction ID en la orden

Para que el postback funcione, el Transaction ID debe quedar guardado en los datos de la orden de VTEX. Esto se hace a través del campo `marketingData` del checkout.

En el checkout de VTEX, agrega un script que lea la cookie y la pase al `orderForm` antes de que el usuario complete la compra:

```html
<script>
  // Leer el Transaction ID de la cookie
  var match = document.cookie.match(/(?:^|; )proof_tid=([^;]+)/);
  var efTid = match ? match[1] : null;

  if (efTid) {
    // Obtener el orderFormId del checkout actual
    var orderFormId = vtxjs.checkout.orderFormId; // VTEX Legacy expone esto globalmente

    fetch('/api/checkout/pub/orderForm/' + orderFormId + '/attachments/marketingData', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        utmSource: 'proof',
        utmCampaign: efTid
      })
    });
  }
</script>
```

<Warning>
  La forma de obtener el `orderFormId` puede variar según tu configuración de VTEX. Tu technical account manager te ayuda a identificar dónde colocar este script en tu checkout.
</Warning>

## Paso 3: Reportar la conversión (Postback S2S)

Configura un webhook en VTEX para que notifique a tu servidor cuando se completa una orden. Tu servidor lee la orden, extrae el Transaction ID y envía la conversión a Proof.

### Configurar el webhook

1. Ve a **VTEX Admin** > **Orders Management** > **Settings** > **Notification**.
2. Configura un endpoint en tu servidor que reciba la notificación de órdenes nuevas.

### Disparar el postback desde tu servidor

Cuando VTEX notifica una nueva orden, tu servidor extrae el Transaction ID de `marketingData.utmCampaign` y envía la conversión:

```javascript
// Disparar postback a Proof con los datos de la orden
const params = new URLSearchParams({
  nid: '3935',
  transaction_id: efTid,        // El Transaction ID guardado en marketingData.utmCampaign
  amount: String(orderTotal),    // Monto de la venta
  adv_event_id: orderId          // Número de orden
});

await fetch(`https://www.track.proof.nilho.com/?${params}`);
```

<Info>
  Recuerda que VTEX almacena los montos en **centavos**. Divide por 100 antes de enviar el `amount` a Proof.
</Info>

## Probar la integración

<Steps>
  <Step title="Genera un tracking link de prueba en Proof">
    Crea un tracking link con un creador de prueba para tu oferta.
  </Step>
  <Step title="Haz clic en el tracking link">
    Abre el link en un navegador en modo incógnito. Verifica que llegas a tu tienda VTEX.
  </Step>
  <Step title="Completa una compra de prueba">
    Completa el checkout con un producto de prueba.
  </Step>
  <Step title="Verifica en Proof">
    En **Reports** > **Conversions**, confirma que la conversión aparece con el monto y número de orden correctos.
  </Step>
</Steps>

## Solución de problemas

<Accordion title="Las conversiones no aparecen en Proof">
  1. Verifica que el webhook de VTEX está configurado y apuntando a tu servidor.
  2. Confirma que el Transaction ID se está guardando en `marketingData.utmCampaign` de la orden.
  3. Revisa los logs de tu servidor para confirmar que el postback se está enviando.
</Accordion>

<Accordion title="El Transaction ID no se guarda en la orden">
  Verifica que el script del checkout está leyendo la cookie `proof_tid` correctamente y que el `orderFormId` es el correcto. Tu technical account manager puede ayudarte a depurar esto.
</Accordion>

<Accordion title="El monto no coincide">
  VTEX maneja montos en centavos. Si el valor en VTEX es `15000000`, divídelo por 100 para obtener `150000.00` antes de enviarlo a Proof.
</Accordion>
