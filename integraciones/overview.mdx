## VTEX: ¿Qué versión tengo?

VTEX tiene tres variantes principales. La integración varía según cuál uses:

| Variante | Descripción | Guía |
|---|---|---|
| **VTEX Legacy (Portal/CMS)** | La plataforma monolítica original. Checkout clásico de VTEX. | [Integrar VTEX Legacy](/integraciones/vtex-legacy) |
| **VTEX IO / Cloud (Store Framework)** | Arquitectura moderna basada en componentes VTEX IO. | [Integrar VTEX Cloud](/integraciones/vtex-cloud) |
| **VTEX Headless (FastStore / Custom)** | Frontend desacoplado (React/Next.js). Comunica con VTEX via APIs. | [Integrar VTEX Headless](/integraciones/vtex-headless) |

<Accordion title="¿Cómo sé qué versión de VTEX tengo?">
  - **VTEX Legacy**: Tu tienda usa el CMS clásico de VTEX con templates HTML. La URL del admin es `{cuenta}.myvtex.com/admin/Site/Template.aspx`.
  - **VTEX IO / Cloud**: Tu tienda se desarrolla con VTEX IO CLI y apps. Usas `vtex link` y `vtex deploy` para desarrollar.
  - **VTEX Headless**: Tu frontend es una app separada (FastStore, Gatsby, Next.js) que consume las APIs de VTEX. No usas el storefront de VTEX directamente.

  Si no estás seguro, pregúntale a tu technical account manager.
</Accordion>
